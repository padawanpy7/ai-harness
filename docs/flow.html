ai-harness / flujo

loop controlado: spec -> compuerta humana -> especialistas (TDD) -> verifier

Loop controlado, no "goal mode". Cada fase es una compuerta: la IA es probabilistica, una cadena larga deriva. Spec primero (SDD), revision humana, especialistas que codean con TDD, y un verifier que opera la app de verdad antes de dar algo por bueno. Si el verifier rechaza, vuelve al especialista.
